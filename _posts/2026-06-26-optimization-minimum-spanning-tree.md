# 📖 故事引入

某乡镇准备为几个村庄铺设光纤网络。

每两个村庄之间都可以铺设光纤，但由于距离不同，施工费用也不一样。

预算有限，不可能把所有村庄之间都修建光纤。

可是，如果有村庄没有接入网络，整个工程又失去了意义。

负责人真正需要回答的问题不是：

> 哪两座村庄之间距离最近？

而是：

> **怎样用最少的建设成本，把所有村庄都连接起来？**

这个问题不仅出现在通信网络建设，还广泛应用于电网铺设、供水管道建设、道路规划等领域。

今天要学习的，就是解决这类问题的经典模型——**最小生成树模型**。

# 🧠 什么是最小生成树？

一句话来说：

> **最小生成树，就是用最小的总代价，把网络中的所有节点连接起来。**

这里需要注意两个关键词：

第一，是**所有节点都要连通**。

如果还有村庄没有接入网络，那么方案就是不可行的。

第二，是**总代价最小**。

只要能够保证所有节点互相连通，就没有必要修建更多道路，因为每增加一条线路，都意味着更多成本。

因此，最小生成树追求的是：

- 不遗漏任何节点；
    
- 不形成重复连接；
    
- 总建设成本最低。
    

它解决的是"连接整个网络"的问题，而不是"寻找一条最短路线"的问题。

# 🚩 什么时候想到最小生成树？

判断是否应该使用最小生成树，可以重点关注下面这些关键词：

- 铺设电网
    
- 建设通信网络
    
- 修建管道
    
- 铺设光纤
    
- 修建道路
    
- 连接所有地点
    
- 建设成本最低
    

这些问题都有一个共同特点：

> **目标不是到达某一个地点，而是让所有地点都连通。**

例如：

- 给所有村庄接通自来水；
    
- 给所有学校铺设校园网络；
    
- 建立整个园区的电力线路。
    

如果题目要求：

> 从 A 点到 B 点最快。

应该想到最短路径。

如果题目要求：

> 把所有地点全部连接起来。

那么就应该优先考虑最小生成树。

一个简单的判断方法就是：

> **是否需要连接整个网络，而不是寻找某一条路线？**

如果答案是肯定的，那么最小生成树就是非常典型的建模选择。

# 🏆 一个完整案例

某乡镇计划为五个村庄铺设光纤。

经过测量，不同村庄之间的施工费用如下：

|起点|终点|建设费用（万元）|
|---|---|--:|
|A|B|8|
|A|C|5|
|B|C|6|
|B|D|7|
|C|D|4|
|C|E|9|
|D|E|3|

今天到底需要决定什么？

不是修建多少公里光纤，而是：

**哪些线路需要建设。**

接下来，还要回答另一个问题：

怎样才算建设成本最低？

如果把所有线路都建设，当然能够连通所有村庄，但费用最高。

如果线路修得太少，又可能导致部分村庄无法联网。

因此，我们需要寻找一种方案：

既能够保证五个村庄全部连通，又让建设费用尽可能低。

整个建模过程依然是在完成"现实问题 → 数学表达"。

例如：

- 一个村庄 → 一个节点；
    
- 一条光纤线路 → 一条边；
    
- 建设费用 → 边的权值；
    
- 所有村庄联网 → 图保持连通。
    

最终，我们需要找到一棵连接所有节点、总代价最小的生成树。

# 💻 Python 实现

Python 中可以利用 `networkx` 快速建立网络，并求解最小生成树。

```python
import networkx as nx

# 创建无向图
G = nx.Graph()

# 添加边（村庄1，村庄2，费用）
edges = [
    ("A", "B", 8),
    ("A", "C", 5),
    ("B", "C", 6),
    ("B", "D", 7),
    ("C", "D", 4),
    ("C", "E", 9),
    ("D", "E", 3)
]

G.add_weighted_edges_from(edges)

# 求最小生成树
mst = nx.minimum_spanning_tree(G)

print("建设线路：")
for u, v, data in mst.edges(data=True):
    print(f"{u} - {v}：{data['weight']} 万元")

total_cost = sum(data["weight"] for _, _, data in mst.edges(data=True))
print("总建设费用：", total_cost, "万元")
```

这里最关键的一行是：

```python
nx.minimum_spanning_tree(G)
```

它对应数学模型中的：

> **寻找连接所有节点且总代价最小的生成树。**

程序最终输出的，就是建议建设的线路以及总建设成本。

# 📌 本章小结

最小生成树关注的不是一条路线，而是整个网络。

学习最小生成树时，可以先问自己三个问题：

1. **是否需要连接所有节点？**
    
2. **连接成本是否不同？**
    
3. **目标是不是让总建设成本最低？**
    

如果这三个问题的答案都是肯定的，那么就可以尝试建立最小生成树模型。

需要特别注意的是：

- **最短路径**解决的是"从一个点到另一个点怎么走"；
    
- **最小生成树**解决的是"怎样把所有点连起来"。
    

虽然它们都建立在图模型上，但解决的问题完全不同。

> **寻找一条路，想到最短路径；连接一张网，想到最小生成树。**
