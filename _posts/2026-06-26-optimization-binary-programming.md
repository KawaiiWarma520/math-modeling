# 📖 故事引入

一家社区便利店准备升级饮品区，希望购买几台新的咖啡设备。

市场上有全自动咖啡机、现磨咖啡机、冰饮机、制冰机等多种设备，每种设备都能提升营业额，但预算有限，不可能全部购买。

老板真正需要回答的问题不是买多少台，而是：

**这台设备买，还是不买？**

每一种设备只有两个状态：要么购买，要么放弃，不存在买半台，也不存在买两台。

像这样的决策，在数学建模中非常常见。

今天要学习的，就是专门解决这类"选与不选"问题的模型——**0-1 整数规划**。

# 🧠 什么是 0-1 整数规划？

一句话来说：

> **0-1 整数规划，就是让每个决策变量只能取 0 或 1 的优化模型。**

在线性规划中，变量可以是任意实数；整数规划要求变量必须是整数；而 0-1 整数规划更进一步，把变量限制为：

$$  
x_i\in{0,1}  
$$

这里：

- $0$ 表示**不选择**；
    
- $1$ 表示**选择**。
    

这种建模方式特别适合描述现实中的"开关型"决策。

例如：

- 是否建设一个仓库；
    
- 是否选择一条运输路线；
    
- 是否投资一个项目；
    
- 是否录用一名候选人；
    
- 是否购买一种设备。
    

很多看似复杂的问题，本质上都是一连串"选"或"不选"组成的决策。

# 🚩 什么时候想到 0-1 整数规划？

识别 0-1 整数规划，可以重点关注题目中的这些表达：

- 是否选择
    
- 是否建设
    
- 是否采购
    
- 是否投资
    
- 是否开放
    
- 是否安排
    
- 最多选择几个
    
- 至少选择几个
    

如果每个方案只能选择一次，那么通常就可以把变量定义为：

$$
x_i = \begin{cases} 
1, & \text{选择} \\ 
0, & \text{不选择} 
\end{cases}
$$

例如题目说：

> 预算有限，从若干项目中选择最优组合。

翻译成数学语言，就是：

- 每个项目对应一个变量；
    
- 选择项目记为 1；
    
- 不选择记为 0。
    

需要注意的是，如果题目允许购买多台设备、建设多个仓库或生产多件产品，那么变量就不再只有 0 和 1，而应采用普通整数规划。

因此，一个简单的判断方法就是：

> **每个方案是不是只有"选"和"不选"两种状态？**

如果答案是肯定的，就可以优先考虑 0-1 整数规划。

# 🏆 一个完整案例

一家社区便利店计划升级咖啡专区。

目前有四种设备可供选择：

|设备|采购费用（万元）|预计每天增加利润（元）|
|---|--:|--:|
|全自动咖啡机|6|900|
|现磨咖啡机|5|750|
|冰饮机|4|520|
|制冰机|3|430|

店长最多只能投入 **12 万元**预算。

今天到底需要决定什么？

不是购买多少台，而是：

每台设备是否购买。

于是定义变量：

$$
x_i = 
\begin{cases} 
1, & \text{购买} \\ 
0, & \text{不购买} 
\end{cases}
$$

怎样才算收益最高？

如果购买全自动咖啡机，就增加 900 元利润；如果购买现磨咖啡机，就增加 750 元利润。

因此，总利润就是：

$$  
\max Z=900x_1+750x_2+520x_3+430x_4  
$$

为什么不能全部购买？

因为预算有限。

购买四种设备分别需要：

6、5、4、3 万元。

因此，总采购费用为：

$$  
6x_1+5x_2+4x_3+3x_4  
$$

预算最多只有 12 万元，因此得到约束：

$$  
6x_1+5x_2+4x_3+3x_4\le12  
$$

最后，每个变量只能取：

$$  
x_i\in{0,1}  
$$

整理之后，完整模型为：

$$
\begin{array}{ll}
\max & 900x_1 + 750x_2 + 520x_3 + 430x_4 \\
\text{s.t.} & \left\{ \begin{array}{l}
6x_1 + 5x_2 + 4x_3 + 3x_4 \le 12, \\
x_i \in \{0, 1\}, \quad i = 1, \cdots, 4.
\end{array} \right.
\end{array}
$$

整个建模过程依然是在完成"现实问题 → 数学表达"的转换。

例如：

- 购买全自动咖啡机 → $x_1=1$
    
- 放弃制冰机 → $x_4=0$
    
- 总预算不超过 12 万元 → $6x_1+5x_2+4x_3+3x_4\le12$
    

不同的是，这一次变量代表的不再是数量，而是**选择**。

# 💻 Python 实现

数学模型建立完成后，可以使用 **PuLP** 建模求解。

变量类型从整数改为二元变量：

```python
cat="Binary"
```

完整代码如下：

```python
from pulp import LpProblem, LpVariable, LpMaximize, lpSum, PULP_CBC_CMD

# 创建最大化问题
model = LpProblem("Coffee_Equipment", LpMaximize)

# 二元变量
x = [LpVariable(f"x{i}", cat="Binary") for i in range(4)]

# 利润和成本
profit = [900, 750, 520, 430]
cost = [6, 5, 4, 3]

# 目标函数
model += lpSum(profit[i] * x[i] for i in range(4))

# 预算约束
model += lpSum(cost[i] * x[i] for i in range(4)) <= 12

# 求解
model.solve(PULP_CBC_CMD(msg=False))

print("是否购买各设备：")
for i, var in enumerate(x, start=1):
    print(f"x{i} =", int(var.value()))

print("最大预计日利润：", model.objective.value(), "元")
```

与整数规划最大的区别在于：

```python
cat="Binary"
```

这意味着变量只能取 **0** 或 **1**，正好对应数学模型中的：

$$  
x_i\in{0,1}  
$$

数学中的"选择或不选择"，在程序中也就变成了一个二元变量。

# 📌 本章小结

0-1 整数规划本质上仍然属于整数规划，但它解决的是一类更加特殊的问题：

**不是决定做多少，而是决定做还是不做。**

学习这一模型时，可以先问自己三个问题：

1. **需要选择哪些方案？**
    
2. **目标是收益最大还是成本最低？**
    
3. **每个方案是不是只能选择一次？**
    

如果第三个问题的答案是肯定的，那么 0-1 整数规划往往就是最自然的建模方法。

从项目投资、设备采购，到仓库选址、路线选择，很多数学建模竞赛题目都可以归结为一系列"选"与"不选"的决策。

> **很多优化问题，看似复杂，最终都可以拆解成一个个简单的选择。**
