# 📖 故事引入

某互联网公司用决策树做用户流失预测。

一开始效果不错，但上线后发现一个问题：

- 有时候预测很准
    
- 有时候结果却波动很大
    
- 换一批数据，模型表现就变了
    

技术团队发现一个本质问题：

> **单棵决策树太“敏感”，容易受数据影响。**

于是他们提出一个更稳妥的想法：

> **如果不是一棵树，而是一片森林呢？**

今天要学习的，就是解决这类问题的经典模型——**随机森林**。

---

# 🧠 什么是随机森林？

一句话来说：

> **随机森林，就是由多棵决策树组成，通过“投票”来决定最终结果的模型。**

核心思想非常简单：

- 一棵树 → 容易偏差
    
- 多棵树 → 平均误差
    
- 多数投票 → 更稳定
    

---

# 🚩 什么时候想到随机森林？

如果题目中出现下面这些情况，就可以考虑随机森林：

- 分类或回归问题
    
- 数据复杂
    
- 特征较多
    
- 结果不稳定
    
- 单模型效果不好
    
- 需要高精度预测
    

例如：

- 用户是否流失
    
- 信用评分
    
- 医疗诊断
    
- 风险预测
    

一个简单判断方法：

> **如果决策树效果不错，但不稳定，就可以升级为随机森林。**

---

# 🧠 随机森林的核心思想

随机森林做了两件关键的“随机化”操作：

---

## ① 数据随机（Bootstrap）

每棵树都用不同的数据子集：

- 有放回抽样
    
- 每棵树看到的数据不同
    

👉 目的：让每棵树“学到不同经验”

---

## ② 特征随机

每个节点分裂时：

- 不看全部特征
    
- 只看部分特征
    

👉 目的：减少“强特征垄断”

---

## ③ 集成投票

分类问题：

$$  
最终结果 = 多数投票  
$$

回归问题：

$$  
最终结果 = 平均值  
$$

---

# 🧠 为什么随机森林更好？

单棵树的问题：

- 容易过拟合
    
- 对数据敏感
    

随机森林解决：

> “用多个不完美模型，降低整体误差”

---

# 🏆 一个完整案例

某公司要预测用户是否流失。

数据如下：

|用户|登录次数|使用时长|是否流失|
|---|--:|--:|---|
|A|10|30|否|
|B|2|5|是|
|C|8|20|否|
|D|3|6|是|
|E|9|25|否|

---

## 决策树问题

单棵树可能：

- 对某个特征过度依赖
    
- 导致误判
    

---

## 随机森林做法

建立多棵树：

- Tree 1：重点看登录次数
    
- Tree 2：重点看使用时长
    
- Tree 3：随机组合特征
    

---

## 最终结果：

> 多数树认为“是否流失”

---

# 💻 Python 实现

```python
import numpy as np
from sklearn.ensemble import RandomForestClassifier

# 特征：登录次数 + 使用时长
X = np.array([
    [10, 30],
    [2, 5],
    [8, 20],
    [3, 6],
    [9, 25]
])

# 标签：是否流失
y = np.array([0, 1, 0, 1, 0])

# 随机森林模型
model = RandomForestClassifier(
    n_estimators=100,
    random_state=0
)

model.fit(X, y)

# 预测
print(model.predict([[4, 10]]))
print(model.predict_proba([[4, 10]]))
```

# 📌 本章小结

随机森林解决的是：

> **单个模型不稳定的问题**

学习随机森林时，可以记住三个关键词：

1. **多模型（多个决策树）**
    
2. **随机性（数据 + 特征）**
    
3. **投票机制（集成结果）**
    

---

## 🔥 和决策树的区别

|模型|特点|
|---|---|
|决策树|可解释，但不稳定|
|随机森林|稳定，但可解释性下降|

---

> **决策树像一个专家，随机森林像一群专家开会。**
