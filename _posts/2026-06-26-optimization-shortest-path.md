# 📖 故事引入

中午十二点，外卖订单不断涌来。

骑手需要从配送站出发，把餐品尽快送到顾客手中。

打开地图后，你会发现，从配送站到顾客家并不只有一条路。

有的路线距离较短，却容易堵车；有的路线虽然稍远，但一路畅通；还有的路线需要经过多个路口。

面对这些选择，骑手真正需要回答的问题不是：

> 哪条路看起来最近？

而是：

> **怎样才能最快到达目的地？**

这个问题不仅出现在外卖配送，还广泛应用于物流运输、导航软件、通信网络等领域。

今天要学习的，就是解决这类问题的经典模型——**最短路径模型**。

# 🧠 什么是最短路径？

一句话来说：

> **最短路径，就是在一张网络中寻找从起点到终点代价最小的一条路线。**

这里的"最短"，并不一定指距离最短。

根据问题不同，它还可以表示：

- 行驶时间最短；
    
- 配送成本最低；
    
- 能耗最小；
    
- 风险最低。
    

因此，我们更喜欢把每条道路看作一条"边"，并赋予它一个"代价"。

例如：

- 一段道路长度为 2 km；
    
- 一条高速需要收费 15 元；
    
- 一条道路预计耗时 8 分钟。
    

这些都可以作为边的权值。

整个城市就可以抽象成一张图。

于是，现实中的导航问题，就变成了图上的最短路径问题。

# 🚩 什么时候想到最短路径？

判断是否应该使用最短路径模型，可以先观察题目中是否出现了"网络结构"。

例如：

- 城市道路；
    
- 地铁线路；
    
- 快递配送；
    
- 电力线路；
    
- 通信网络；
    
- 管道运输。
    

如果问题要求：

- 从 A 到 B 怎样最快；
    
- 怎样花费最少；
    
- 怎样经过若干节点；
    

那么通常都可以建立图模型。

建立图模型后，再把：

- 路口看作**节点**；
    
- 道路看作**边**；
    
- 距离、时间或费用看作**边权**。
    

这样，现实问题就完成了第一次数学抽象。

需要注意的是，如果题目要求访问多个地点并最终返回起点，那么它更可能属于**旅行商问题（TSP）**，而不是普通最短路径。

因此，一个简单的判断方法就是：

> **是否只需要寻找两个节点之间的一条最优路线？**

如果答案是肯定的，就可以优先考虑最短路径模型。

# 🏆 一个完整案例

某外卖配送站需要把一份订单送到顾客家。

配送站和道路可以抽象成下面的网络：

|道路|行驶时间（分钟）|
|---|--:|
|配送站 → A|4|
|配送站 → B|2|
|A → C|5|
|B → C|3|
|B → D|6|
|C → 顾客|4|
|D → 顾客|2|

今天到底需要决定什么？

不是决定配送多少单，而是：

**选择一条配送路线。**

接下来，还要回答另一个问题：

怎样才算最快？

假设每条道路的数字表示预计行驶时间，那么整条路线的耗时，就是经过所有道路时间的总和。

例如：

配送站 → B → C → 顾客

总耗时就是：

$$  
2+3+4=9\text{（分钟）}  
$$

而另一条路线：

配送站 → A → C → 顾客

耗时为：

$$  
4+5+4=13\text{（分钟）}  
$$

显然，第二条路线更慢。

因此，我们真正需要寻找的是：

> **所有可行路线中，总耗时最小的一条。**

整个建模过程依然是在完成"现实问题 → 数学表达"。

例如：

- 一个路口 → 一个节点；
    
- 一条道路 → 一条边；
    
- 行驶时间 → 边的权值；
    
- 最快送达 → 最小路径长度。
    

经过这样的抽象，现实中的配送问题就转换成了图论中的最短路径问题。

# 💻 Python 实现

求解最短路径，最经典的方法之一就是 **Dijkstra 算法**。

下面使用 `networkx` 建立道路网络。

```python
import networkx as nx

# 创建有向图
G = nx.DiGraph()

# 添加道路（起点，终点，时间）
edges = [
    ("配送站", "A", 4),
    ("配送站", "B", 2),
    ("A", "C", 5),
    ("B", "C", 3),
    ("B", "D", 6),
    ("C", "顾客", 4),
    ("D", "顾客", 2)
]

G.add_weighted_edges_from(edges)

# 求最短路径
path = nx.shortest_path(G, "配送站", "顾客", weight="weight")
distance = nx.shortest_path_length(G, "配送站", "顾客", weight="weight")

print("最短路径：", " → ".join(path))
print("总耗时：", distance, "分钟")
```

这里最关键的一行是：

```python
nx.shortest_path(...)
```

它对应数学模型中的：

> **寻找总代价最小的一条路径。**

当图建立完成后，求解器会自动计算从起点到终点的最优路线。

# 📌 本章小结

最短路径模型解决的，不是"距离有多远"，而是**怎样从一个地方到另一个地方最优**。

学习最短路径时，可以先问自己三个问题：

1. **问题能否抽象成一张网络？**
    
2. **节点之间是否存在连接关系？**
    
3. **是否需要寻找两个节点之间的最优路线？**
    

如果这三个问题的答案都是肯定的，那么就可以尝试建立最短路径模型。

从地图导航、外卖配送，到物流运输、通信网络，很多现实问题都可以抽象成一张图。而数学建模的重要一步，就是把现实中的道路、站点和时间，翻译成节点、边和权值。

> **当现实能够画成一张图，最短路径往往就是第一个值得想到的模型。**
