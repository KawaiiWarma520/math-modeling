# 数学建模知识分享网站

## 项目概况
- 网站地址：https://kawaiiwarma520.github.io/math-modeling/
- 技术栈：Jekyll + GitHub Pages
- 定位：一位大二学生的数学建模学习笔记与竞赛经验分享

## 栏目结构（共 6 个）
1. 🌟 新手教程 — 零基础入门
2. 📚 模型百科 — 六大类模型原理与代码
3. ✍️ 论文写作 — 论文框架、摘要写法等
4. 🛠 工具学习 — LaTeX、Python、VS Code 等
5. 📝 真题演练 — 历年真题 + 解题思路
6. 📁 资料分享 — 模板、代码库、配置文件等

## 内容处理规范（重要）

### 用户原创内容
- **完整保留用户原始内容**，不做删减
- 用户的原始文件（Obsidian 笔记等）是最终依据
- 不允许移除"学术写作"（论文写法）、"应用场景"、"注意事项"、代码实现等完整章节
- 不允许替换示例数据或改变代码语言（如 Python ↔ MATLAB）
- 允许移除英文副标题如 `(Core Concept)`，其余内容不变

### 文章格式规范
- 每篇文章必须包含完整的 Jekyll Front Matter：
  ```yaml
  layout: post
  title: "标题"
  date: YYYY-MM-DD HH:MM:SS +0800
  categories: 栏目名
  tags: [标签1, 标签2]
  description: "一段有吸引力的摘要"
  ```
- 日期使用实际日期，不使用占位日期
- description 字段必须填写有意义的摘要，不显示文章第一句话

### 技术规范
- 所有涉及数学公式的文章必须确保 KaTeX 渲染正常
- 搜索功能必须包含 `.catch()` 错误处理和加载状态
- 标签页锚点跳转需添加 `scroll-margin-top` 防止被导航栏遮挡
- 标签显示不使用 emoji，用纯文字即可

### 工作流程
1. 用户提供原始内容
2. 按格式整理，完整保留内容
3. 发回给用户审阅
4. 用户确认后再推送上线
5. 推送前检查：内容完整性、日期、LaTeX、搜索功能
