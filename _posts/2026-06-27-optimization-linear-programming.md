# 📖 故事引入

假如你经营着一家面包店。

每天清晨，面包师都会问你一个问题：今天应该烤多少吐司？又该做多少可颂？

如果吐司利润高，就一直做吐司吗？显然不行，因为面粉、黄油、烤箱时间和员工工时都是有限的。

如果平均分配资源呢？又未必能赚到最多的钱。

你真正需要回答的问题其实只有一个：

> **有限的资源，到底应该怎样分配，才能获得最大的收益？**

这个问题不仅出现在面包店，在餐厅排班、农田种植、广告投放、物流配送等场景中都会遇到。

今天要学习的，就是解决这类问题最经典的模型——**线性规划**。

# 🧠 什么是线性规划？

一句话来说：

> **线性规划，就是在有限资源下寻找最优决策的方法。**

现实中的很多问题，都需要我们做出选择。

例如：

- 今天生产多少产品？
- 怎样安排运输路线？
- 每种原料采购多少？
- 广告预算如何分配？

这些问题虽然背景不同，但都可以归结为三个核心问题：

- **需要决定什么？**
- **希望达到什么目标？**
- **受到哪些限制？**

在线性规划中，我们把需要决定的量称为**决策变量**；希望最大化或最小化的指标称为**目标函数**；现实中的各种限制，则写成**约束条件**。

之所以叫"线性"，是因为目标函数和约束条件都可以写成变量的一次表达式，例如：

$$Z = 15x_1 + 18x_2$$

或者

$$2x_1 + 4x_2 \le 60$$

当现实问题能够翻译成这样的数学表达式时，就可以考虑使用线性规划进行求解。

# 🚩 什么时候想到线性规划？

很多同学学习模型时，总是在看到题目后不知道该用什么方法。

其实，识别模型往往比求解模型更重要。

如果题目中频繁出现下面这些关键词，就应该优先想到线性规划：

- 利润最大
- 成本最低
- 收益最高
- 时间最短
- 资源有限
- 合理分配
- 最优方案
- 预算限制

除此之外，还可以观察题目的结构。

如果题目要求你**决定每种方案的数量**，同时又受到多种资源限制，例如资金、时间、人力、原材料等，那么大概率就是一个优化问题。

接着再判断目标函数和约束条件是否都是线性的。

例如，假设我们有两种产品：吐司（$x_1$）和蛋糕（$x_2$）。

题目说：

> 一个吐司需要 2 kg 面粉，一个蛋糕需要 4 kg 面粉。

这句话可以直接翻译为：

$$2x_1 + 4x_2$$

题目说：

> 一天最多只有 60 kg 面粉。

于是得到约束：

$$2x_1 + 4x_2 \le 60$$

可以看到，数学建模真正做的事情，并不是写公式，而是把一句普通的话翻译成数学表达。

当然，并不是所有优化问题都适合使用线性规划。

如果变量必须取整数，就需要考虑整数规划；如果目标函数包含平方项、指数项等非线性表达式，则属于非线性规划；如果决策之间具有明显的时间递推关系，则动态规划往往更加合适。

因此，在建模之前，先判断问题是否满足"线性"这一前提，是十分重要的一步。

# 🏆 一个完整案例

假设一家社区面包店每天只生产两种产品：

- 吐司
- 可颂

老板每天需要决定生产多少个，才能获得最高利润。

已知：

- 一个吐司利润 15 元。
- 一个可颂利润 18 元。

但是资源有限。

每天最多只有：

- 面粉 60 kg；
- 黄油 30 kg；
- 烘焙时间 40 小时。

制作一个吐司需要：

- 面粉 2 kg；
- 黄油 2 kg；
- 烘焙时间 3 小时。

制作一个可颂需要：

- 面粉 4 kg；
- 黄油 1 kg；
- 烘焙时间 2 小时。

今天到底需要决定什么？

答案就是：

- 吐司生产多少个，用变量 $x_1$ 表示；
- 可颂生产多少个，用变量 $x_2$ 表示。

这一步就是建立决策变量。

接下来，我们希望什么？

当然是利润最高。

每生产一个吐司增加 15 元利润，每生产一个可颂增加 18 元利润，因此总利润可以写成：

$$\max Z = 15x_1 + 18x_2$$

然后，还需要回答另一个问题：为什么不能一直生产？

因为资源会耗尽。

面粉约束：吐司需要 2 kg，可颂需要 4 kg，总消耗为 $2x_1 + 4x_2$，上限 60 kg：

$$2x_1 + 4x_2 \le 60$$

黄油约束：吐司需要 2 kg，可颂需要 1 kg，总消耗为 $2x_1 + x_2$，上限 30 kg：

$$2x_1 + x_2 \le 30$$

烘焙时间约束：吐司 3 小时，可颂 2 小时，总时间为 $3x_1 + 2x_2$，上限 40 小时：

$$3x_1 + 2x_2 \le 40$$

最后，不可能生产负数量，因此还有：

$$x_1 \ge 0, \quad x_2 \ge 0$$

把这些内容整理在一起，一个完整的线性规划模型就建立完成了：

$$
\begin{aligned}
& \max \; Z = 15x_1 + 18x_2 \\
& \text{s.t.} \;
\begin{cases}
2x_1 + 4x_2 \leq 60 \\
2x_1 + x_2 \leq 30 \\
3x_1 + 2x_2 \leq 40 \\
x_1 \geq 0, \; x_2 \geq 0
\end{cases}
\end{aligned}
$$

回顾整个建模过程，你会发现，我们几乎没有做复杂的数学推导，而是在不断完成一件事情：

> 现实问题 → 数学表达

例如：

- 一个吐司需要 2 kg 面粉 → $2x_1$
- 一个可颂需要 4 kg 面粉 → $4x_2$
- 每天只有 60 kg 面粉 → $2x_1 + 4x_2 \le 60$
- 每个可颂利润 18 元 → $18x_2$

这正是数学建模最核心的能力。

# 💻 Python 实现

建立数学模型之后，就可以交给计算机求解。

目标函数为：

$$15x_1 + 18x_2$$

由于 `scipy.optimize.linprog()` 默认求最小值，因此需要把目标函数取相反数：

```python
c = [-15, -18]
```

完整代码如下：

```python
import numpy as np
from scipy.optimize import linprog

# 目标函数（取负号表示求最大值）
c = [-15, -18]

# 约束矩阵（每一行对应一个 ≤ 约束的左边系数）
A = [
    [2, 4],   # 面粉：2x₁ + 4x₂ ≤ 60
    [2, 1],   # 黄油：2x₁ + x₂ ≤ 30
    [3, 2]    # 烘焙时间：3x₁ + 2x₂ ≤ 40
]

# 约束右端（每个资源的上限）
b = [60, 30, 40]

# 决策变量范围（非负）
bounds = [(0, None), (0, None)]

# 求解线性规划
result = linprog(
    c=c,
    A_ub=A,
    b_ub=b,
    bounds=bounds,
    method="highs"
)

# 输出结果
if result.success:
    print("✅ 求解成功！")
    print(f"🥖 吐司生产数量：{result.x[0]:.2f} 个")
    print(f"🥐 可颂生产数量：{result.x[1]:.2f} 个")
    print(f"💰 最大利润：{-result.fun:.2f} 元")
    
    # 资源使用情况分析
    print("\n📊 资源使用情况：")
    flour_used = 2 * result.x[0] + 4 * result.x[1]
    butter_used = 2 * result.x[0] + 1 * result.x[1]
    time_used = 3 * result.x[0] + 2 * result.x[1]
    print(f"面粉使用：{flour_used:.2f} / 60 kg")
    print(f"黄油使用：{butter_used:.2f} / 30 kg")
    print(f"烘焙时间：{time_used:.2f} / 40 小时")
else:
    print("❌ 求解失败，请检查模型设置。")
```

需要注意的是，线性规划得到的结果可能包含小数。如果题目要求产品数量、车辆数量或人员数量必须为整数，就不能直接使用普通线性规划，而应进一步建立整数规划模型（例如使用 `pulp` 或 `scipy.optimize.milp`）。

# 📌 本章小结

学习线性规划，真正重要的并不是记住某个公式，而是养成一种建模思维。

面对一道题目，可以先问自己三个问题：

1. **什么需要我决定？** → 定义决策变量
2. **希望优化什么？** → 写出目标函数
3. **有哪些资源限制？** → 列出约束条件

当这三个问题都能回答清楚，并且目标函数和约束条件都能够写成线性表达式时，一个线性规划模型就已经呼之欲出了。

从面包店的生产计划，到物流运输、农业种植、投资组合，再到数学建模竞赛中的资源分配问题，线性规划解决的始终都是同一类问题：**在有限资源下寻找最优决策。**

> **数学建模不是套公式，而是把现实问题翻译成数学语言。**
