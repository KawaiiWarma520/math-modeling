# 📖 故事引入

午餐高峰期，一家外卖平台同时收到四份订单。

与此同时，附近也正好有四位骑手等待接单。

由于每位骑手所在的位置不同，到各个商家的距离也不同。

如果随意派单，有些骑手可能需要绕很远的路，而有些骑手却离商家很近。

平台真正需要回答的问题不是：

> 谁先接单？

而是：

> **怎样安排骑手和订单，才能让整体配送效率最高？**

类似的问题，在员工排班、司机派车、机器加工等场景中都非常常见。

今天要学习的，就是解决这类问题的经典模型——**指派问题**。

# 🧠 什么是指派问题？

一句话来说：

> **指派问题，就是把若干任务分配给若干对象，使总成本最低或总收益最高。**

这里有两个角色：

- 一类是**执行者**，例如骑手、员工、机器；
    
- 一类是**任务**，例如订单、工作、加工任务。
    

每一个执行者完成不同任务，成本可能不同。

例如：

- 骑手 A 到订单 1 只需要 3 分钟；
    
- 骑手 A 到订单 2 却需要 10 分钟。
    

因此，不同的分配方案，最终的总成本也不同。

指派问题的目标，就是找到一种最优匹配方式。

# 🚩 什么时候想到指派问题？

如果题目中出现下面这些关键词，就可以优先考虑指派问题：

- 分配任务
    
- 派单
    
- 排班
    
- 指定负责人
    
- 分配机器
    
- 安排司机
    
- 一一对应
    

这些问题有一个共同特点：

> **每个任务只能分配给一个对象，每个对象也只能负责一个任务。**

例如：

- 四位老师监考四个考场；
    
- 四辆车配送四条线路；
    
- 五台机器加工五个零件。
    

如果题目允许一个人完成多个任务，那么就不再是经典指派问题，而更接近调度问题。

因此，一个简单的判断方法就是：

> **是否需要建立"一对一"的最优匹配？**

如果答案是肯定的，那么就可以考虑建立指派模型。

# 🏆 一个完整案例

一家外卖平台需要安排四位骑手配送四份订单。

根据地图测算，不同骑手完成不同订单所需时间如下：

|骑手|订单1|订单2|订单3|订单4|
|---|--:|--:|--:|--:|
|A|5|8|7|6|
|B|6|4|3|7|
|C|8|5|6|4|
|D|7|6|5|5|

今天到底需要决定什么？

答案就是：

**每位骑手负责哪一份订单。**

接下来，还要回答另一个问题：

怎样才算效率最高？

如果我们把配送时间作为成本，那么所有骑手完成配送所花费的总时间越少，方案就越优。

因此，总成本可以表示为：

$$  
\min Z=\sum_{i=1}^{4}\sum_{j=1}^{4}c_{ij}x_{ij}  
$$

其中：

- $c_{ij}$ 表示骑手 $i$ 完成订单 $j$ 的配送时间；
    
- $x_{ij}$ 表示是否安排骑手 $i$ 配送订单 $j$。
    

这里：

$$  
x_{ij}=  
\begin{cases}  
1,&\text{安排配送}\  
0,&\text{不安排}  
\end{cases}  
$$

为什么不能把所有订单都交给最快的骑手？

因为每位骑手一次只能完成一份订单。

因此，需要满足两个约束：

每位骑手只能配送一份订单：

$$  
\sum_{j=1}^{4}x_{ij}=1  
$$

每份订单只能由一位骑手完成：

$$  
\sum_{i=1}^{4}x_{ij}=1  
$$

整个建模过程依然是在完成：

现实问题

↓

数学表达

例如：

- 一个骑手 → 一行；
    
- 一份订单 → 一列；
    
- 是否分配 → 一个 0-1 变量。
    

最终，我们需要寻找总配送时间最短的匹配方案。

# 💻 Python 实现

Python 中可以利用 `scipy.optimize.linear_sum_assignment()` 求解指派问题。

```python
import numpy as np
from scipy.optimize import linear_sum_assignment

# 配送时间矩阵
cost = np.array([
    [5, 8, 7, 6],
    [6, 4, 3, 7],
    [8, 5, 6, 4],
    [7, 6, 5, 5]
])

# 求最优匹配
row, col = linear_sum_assignment(cost)

print("分配方案：")
for r, c in zip(row, col):
    print(f"骑手 {chr(65+r)} -> 订单 {c+1}")

print("总配送时间：", cost[row, col].sum(), "分钟")
```

这里最关键的一行是：

```python
linear_sum_assignment(cost)
```

它对应数学模型中的目标：

> **寻找总代价最小的一对一匹配。**

程序最终会输出每位骑手对应的订单，以及整个配送任务的最短总时间。

# 📌 本章小结

指派问题关注的不是路线规划，也不是资源分配，而是**如何建立最优的一对一匹配关系**。

学习指派问题时，可以先问自己三个问题：

1. **是否有两类对象需要匹配？**
    
2. **每个对象是否只能匹配一次？**
    
3. **目标是不是让总成本最低或总收益最高？**
    

如果这三个问题的答案都是肯定的，那么就可以尝试建立指派模型。

回顾最近几篇网络优化模型，你会发现：

- **最短路径**：寻找最优路线。
    
- **最小生成树**：连接整个网络。
    
- **最大流**：提高网络运输能力。
    
- **指派问题**：建立最优匹配关系。
    

虽然它们都属于优化模型，但关注的问题各不相同。

> **不是所有优化都是"怎么走"，有些优化更关心"谁去做"。**
