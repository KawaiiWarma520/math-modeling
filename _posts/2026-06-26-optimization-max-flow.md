# 📖 故事引入

一家物流公司准备把仓库里的货物运往城市配送中心。

地图上已经规划好了运输路线，每条道路都有自己的运输能力。有的道路宽阔，一小时可以运输很多货物；有的道路狭窄，只能缓慢通行。

负责人发现，即使继续增加货物，也不一定能够全部送出去。

因为真正限制运输能力的，并不是仓库里的货物数量，而是网络中那些"最拥堵"的道路。

于是，一个新的问题出现了：

> **在现有运输网络下，每小时最多能够运输多少货物？**

今天要学习的，就是解决这类问题的经典模型——**最大流模型**。

# 🧠 什么是最大流？

一句话来说：

> **最大流，就是在满足每条线路容量限制的前提下，寻找网络能够运输的最大流量。**

与最短路径、最小生成树不同，最大流关注的不是路线，也不是建设成本，而是**运输能力**。

在模型中：

- 仓库、物流中心等抽象成**节点**；
    
- 道路、管道、线路抽象成**边**；
    
- 每条边都有一个**容量**，表示单位时间内能够运输的最大数量。
    

最终，我们希望找到一种运输方案，使源点到汇点的总流量达到最大。

# 🚩 什么时候想到最大流？

如果题目出现下面这些关键词，就可以优先考虑最大流模型：

- 最大运输能力
    
- 最大供水量
    
- 最大供电能力
    
- 最大带宽
    
- 最大配送能力
    
- 网络容量
    
- 管道运输
    

这些问题有一个共同特点：

> **每条线路都有容量限制。**

例如：

- 一条道路每小时最多通过 100 辆车；
    
- 一根输水管每分钟最多输送 50 吨水；
    
- 一条网络链路每秒最多传输 1 Gbps 数据。
    

如果题目希望：

> 从起点到终点最多能够运输多少。

那么最大流就是非常典型的建模选择。

需要注意的是，如果题目要求寻找最快路线，应想到最短路径；如果要求连接整个网络，应想到最小生成树；如果要求计算运输能力，则应想到最大流。

一个简单的判断方法就是：

> **网络中的每条边，是否都存在容量上限？**

如果答案是肯定的，那么可以优先考虑最大流模型。

# 🏆 一个完整案例

某物流园需要把货物从总仓库运往配送中心。

运输网络如下：

|起点|终点|最大运输能力（吨/小时）|
|---|---|--:|
|仓库|A|10|
|仓库|B|8|
|A|C|6|
|A|D|4|
|B|D|8|
|C|配送中心|6|
|D|配送中心|10|

今天到底需要决定什么？

不是选择哪条路线，而是：

**每条道路应该运输多少货物。**

接下来，还要回答另一个问题：

为什么不能把所有货物都走同一条路？

因为每条道路都有运输能力上限。

例如：

仓库到 A 的道路最多运输：

$$  
10\ \text{吨/小时}  
$$

即使仓库还有更多货物，也不能继续增加。

这就是容量约束。

与此同时，在中间节点，还必须满足另一个重要条件：

> **流入多少，就必须流出多少。**

例如，对于节点 D：

如果流入 D 的货物为 8 吨，那么流出 D 的货物也必须是 8 吨（不考虑仓储）。

这就是流量守恒。

整个建模过程依然是在完成"现实问题 → 数学表达"。

例如：

- 一条运输线路 → 一条边；
    
- 每小时最多运输 8 吨 → 边容量为 8；
    
- 中转仓不生产也不消耗货物 → 流量守恒。
    

最终，我们需要寻找一种流量分配方案，使从仓库到配送中心的总运输能力达到最大。

# 💻 Python 实现

Python 可以利用 `networkx` 快速建立流网络，并求解最大流。

```python
import networkx as nx

# 创建有向图
G = nx.DiGraph()

# 添加容量（起点，终点，容量）
G.add_edge("仓库", "A", capacity=10)
G.add_edge("仓库", "B", capacity=8)
G.add_edge("A", "C", capacity=6)
G.add_edge("A", "D", capacity=4)
G.add_edge("B", "D", capacity=8)
G.add_edge("C", "配送中心", capacity=6)
G.add_edge("D", "配送中心", capacity=10)

# 求最大流
flow_value, flow_dict = nx.maximum_flow(
    G,
    "仓库",
    "配送中心"
)

print("最大运输能力：", flow_value, "吨/小时")

print("\n各线路运输量：")
for u in flow_dict:
    for v, f in flow_dict[u].items():
        if f > 0:
            print(f"{u} -> {v}: {f}")
```

这里最关键的一行是：

```python
nx.maximum_flow(...)
```

它对应数学模型中的目标：

> **寻找满足所有容量约束和流量守恒条件下的最大运输能力。**

程序不仅会计算网络的最大流量，还会给出每条线路应该承担多少运输任务。

# 📌 本章小结

最大流模型关注的不是路线长短，而是网络整体的运输能力。

学习最大流时，可以先问自己三个问题：

1. **是否存在一个起点和一个终点？**
    
2. **每条线路是否都有容量限制？**
    
3. **目标是不是让总运输能力达到最大？**
    

如果这三个问题的答案都是肯定的，那么就可以尝试建立最大流模型。

回顾最近三篇图论模型，你会发现它们虽然都建立在"图"的基础上，却解决了完全不同的问题：

- **最短路径**：寻找一条最优路线。
    
- **最小生成树**：连接整个网络，建设成本最低。
    
- **最大流**：充分利用现有网络，提高运输能力。
    

真正的建模能力，不只是会求解，更重要的是能够根据问题特点，选择正确的模型。

> **同样是一张图，不同的问题，答案可能完全不同。**
