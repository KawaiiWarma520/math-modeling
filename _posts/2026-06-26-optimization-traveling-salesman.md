# 📖 故事引入

每天上午，快递员都需要到多个小区收件。

这些小区的位置已经确定，收完所有包裹后，还需要返回快递站进行分拣。

如果随意安排路线，很可能会在城市里来回绕路，不仅浪费时间，还增加了油耗。

于是，快递员真正需要回答的问题变成了：

> **怎样才能把所有小区都走一遍，并且最后回到出发点，总路程最短？**

这个问题不仅出现在物流配送，还广泛应用于巡检机器人、无人机巡航、垃圾清运、设备巡检等场景。

今天要学习的，就是组合优化中最经典的问题之一——**旅行商问题（TSP）**。

# 🧠 什么是旅行商问题？

一句话来说：

> **旅行商问题，就是寻找一条经过所有节点且仅经过一次，并最终回到起点的最短路线。**

旅行商问题与最短路径很像，但它们解决的是完全不同的问题。

最短路径只需要回答：

> 从 A 到 B 怎么走？

而旅行商问题需要回答：

> **怎样依次访问所有地点，再返回起点？**

因此，它关注的是一条完整的巡回路线，而不是两点之间的一段路径。

随着地点数量不断增加，可供选择的路线数量会迅速增长，因此旅行商问题也是组合优化领域最著名的问题之一。

# 🚩 什么时候想到旅行商问题？

如果题目中出现下面这些关键词，就应该优先想到旅行商问题：

- 巡检所有设备
    
- 拜访所有客户
    
- 配送所有站点
    
- 收集所有样本
    
- 巡逻所有区域
    
- 最后返回起点
    

这些问题有一个共同特点：

> **所有地点都必须访问，而且通常只访问一次。**

例如：

- 快递员收完所有快件后返回站点；
    
- 无人机巡检所有风机后返回机场；
    
- 检修人员检查所有变电站后回到维修中心。
    

需要注意的是，如果题目只要求从一个地点到另一个地点，那么应该使用最短路径；如果只要求连接所有地点，那么应该使用最小生成树。

一个简单的判断方法就是：

> **是否需要走遍所有节点，并最终形成一个闭合路线？**

如果答案是肯定的，那么就可以考虑建立旅行商问题模型。

# 🏆 一个完整案例

某快递站需要安排一名快递员前往四个社区收件。

所有收件完成后，还需要返回快递站。

根据地图测量，各地点之间的距离如下（单位：km）：

|地点|快递站|A|B|C|D|
|---|--:|--:|--:|--:|--:|
|快递站|0|3|6|7|5|
|A|3|0|4|6|3|
|B|6|4|0|2|5|
|C|7|6|2|0|4|
|D|5|3|5|4|0|

今天到底需要决定什么？

答案就是：

**所有地点的访问顺序。**

接下来，还要回答另一个问题：

怎样才算总路程最短？

如果路线是：

快递站 → A → B → C → D → 快递站

那么总路程就是：

$$  
3+4+2+4+5=18\text{ km}  
$$

如果换一种访问顺序，总路程就会发生变化。

因此，我们真正需要寻找的是：

> **所有可能路线中，总距离最短的一条。**

整个建模过程依然是在完成：

现实问题

↓

数学表达

例如：

- 一个社区 → 一个节点；
    
- 两地之间的距离 → 边权；
    
- 收件顺序 → 路径顺序；
    
- 返回快递站 → 闭合回路。
    

旅行商问题的关键就在于：**既不能遗漏任何地点，也不能重复访问同一个地点。**

# 💻 Python 实现

对于规模较小的问题，可以直接枚举所有可能路线。

下面利用 `itertools.permutations` 演示旅行商问题的基本求解思路。

```python
from itertools import permutations

# 距离矩阵
dist = [
    [0, 3, 6, 7, 5],
    [3, 0, 4, 6, 3],
    [6, 4, 0, 2, 5],
    [7, 6, 2, 0, 4],
    [5, 3, 5, 4, 0]
]

n = len(dist)
best_route = None
best_distance = float("inf")

# 枚举所有访问顺序（0表示快递站）
for route in permutations(range(1, n)):
    path = (0,) + route + (0,)
    total = 0

    for i in range(len(path) - 1):
        total += dist[path[i]][path[i + 1]]

    if total < best_distance:
        best_distance = total
        best_route = path

print("最优路线：", best_route)
print("最短距离：", best_distance, "km")
```

需要注意的是，这种方法适合地点数量较少的情况。

当地点达到几十个、上百个时，路线数量会急剧增加，枚举所有方案几乎不可能完成。这也是为什么旅行商问题常常需要借助遗传算法、模拟退火、蚁群算法等智能优化方法。

# 📌 本章小结

旅行商问题解决的不是"怎样到达一个地点"，而是**怎样高效地访问所有地点**。

学习旅行商问题时，可以先问自己三个问题：

1. **是否需要访问所有地点？**
    
2. **每个地点是否只能访问一次？**
    
3. **最后是否需要回到起点？**
    

如果这三个问题的答案都是肯定的，那么就可以考虑建立旅行商问题模型。

回顾图论优化部分，我们已经学习了五类经典问题：

- **最短路径**：寻找两点之间的最优路线。
    
- **最小生成树**：用最低成本连接整个网络。
    
- **最大流**：提高网络的运输能力。
    
- **指派问题**：建立最优的一对一匹配。
    
- **旅行商问题**：规划访问所有节点的最优巡回路线。
    

虽然它们都建立在图模型上，但解决的问题各不相同。

> **真正困难的不是求路线，而是在所有可能路线中找到最优的一条。**
