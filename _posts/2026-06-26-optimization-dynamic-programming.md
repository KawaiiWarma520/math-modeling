# 📖 故事引入

暑假到了，小李准备来一次五天四夜的旅行。

出发前，他列出了一份心仪景点清单。有的景点门票便宜但体验一般，有的景点风景绝佳，却需要花费更多预算和体力。

问题在于，每天的选择都会影响后面的安排。

如果第一天就把预算花得差不多，后面几天可能只能安排普通景点；如果前两天体力消耗太大，最后几天即使有时间，也很难继续游玩。

这时候，我们面对的已经不是"今天去哪里最好"，而是"怎样规划整个旅程才最好"。

今天要学习的，就是解决这类多阶段决策问题的经典模型——**动态规划**。

# 🧠 什么是动态规划？

一句话来说：

> **动态规划，就是把一个复杂的多阶段决策问题，拆分成一系列相互关联的小问题，再逐步求出整体最优解的方法。**

在线性规划中，我们通常一次性决定所有变量。

而动态规划不同。

它认为，一个复杂问题往往由多个阶段组成，每个阶段都需要做出一次决策，而这次决策又会影响下一阶段。

因此，动态规划通常包含四个要素：

- 阶段：决策进行到哪一步；
    
- 状态：当前处于什么情况；
    
- 决策：这一阶段应该怎样选择；
    
- 状态转移：当前决策会怎样影响下一阶段。
    

动态规划最重要的思想不是"一次求解"，而是**利用前面的最优结果，帮助后面的决策**。

# 🚩 什么时候想到动态规划？

很多同学容易把动态规划和普通优化模型混淆。

其实，一个很简单的问题就能帮助我们判断：

> **当前的决策，会不会影响后面的决策？**

如果答案是肯定的，就应该考虑动态规划。

例如：

- 连续几天的生产计划；
    
- 多天旅行路线安排；
    
- 分阶段投资决策；
    
- 水库调度；
    
- 库存管理；
    
- 能源调度。
    

这些问题都有一个共同特点：

今天的选择，会改变明天能够选择的方案。

如果题目中出现下面这些关键词，也可以优先想到动态规划：

- 第一天、第二天……
    
- 第一步、下一步……
    
- 分阶段决策
    
- 剩余预算
    
- 剩余资源
    
- 后续影响
    

需要注意的是，如果所有决策都是一次性完成，阶段之间没有联系，那么通常不需要使用动态规划，而应优先考虑线性规划或其他优化模型。

# 🏆 一个完整案例

小李计划进行五天旅行。

出发前，他准备了 **3000 元预算**。

每天都有两个景点可以选择：

- 普通景点：花费较少，体验值一般；
    
- 热门景点：体验值更高，但花费更多。
    

今天需要决定什么？

不是一次决定整个旅程，而是：

**每天应该选择哪一种景点。**

接下来，还要回答另一个问题：

为什么今天的选择会影响明天？

因为预算是有限的。

假设第三天已经花掉了大部分预算，那么第四天能够选择的景点就会减少。

因此，在动态规划中，我们可以把：

- 第几天，作为**阶段**；
    
- 剩余预算，作为**状态**。
    

例如：

设

$$  
f(i,b)  
$$

表示：

> **旅行进行到第 $i$ 天，剩余预算为 $b$ 时，能够获得的最大体验值。**

如果今天选择普通景点，那么体验值增加 8 分，预算减少 300 元。

如果今天选择热门景点，那么体验值增加 15 分，预算减少 700 元。

于是，就可以得到状态转移关系：

$$
f(i,b) = \max \begin{cases} 
8 + f(i+1, b-300), & \text{若 } b \ge 300 \\ 
15 + f(i+1, b-700), & \text{若 } b \ge 700 
\end{cases}
$$
$$
\text{边界条件：} f(6, b) = 0 \quad (\text{即第 6 天旅程结束，不再增加体验值}).
$$

这个公式表达的意思很简单：

今天有两种选择。

分别计算未来能够获得的总体验值，再选择更大的那个。

可以看到，我们依然是在完成"现实问题 → 数学表达"的转换。

例如：

- 今天去普通景点 → 预算减少 300 元；
    
- 今天去热门景点 → 预算减少 700 元；
    
- 今天的决定 → 改变明天的预算状态。
    

与前面的优化模型相比，动态规划最大的特点就是：**每一步决策都会影响下一步。**

# 💻 Python 实现

动态规划通常使用数组保存已经计算出的最优结果，避免重复计算。

下面以旅行规划为例，演示如何利用递归和记忆化搜索实现动态规划。

```python
from functools import lru_cache

# 普通景点和热门景点的信息
cost = [300, 700]
score = [8, 15]

days = 5

@lru_cache(maxsize=None)
def dp(day, budget):
    # 五天结束
    if day == days:
        return 0

    best = 0

    # 尝试两种选择
    for c, s in zip(cost, score):
        if budget >= c:
            best = max(best, s + dp(day + 1, budget - c))

    return best

# 初始预算3000元
print("最大体验值：", dp(0, 3000))
```

程序中的：

```python
dp(day, budget)
```

正对应数学模型中的：

$$  
f(i,b)  
$$

函数返回的，就是当前阶段、当前预算下能够获得的最大体验值。

这也是动态规划最经典的实现方式之一。

# 📌 本章小结

动态规划最大的特点，不是优化目标发生了变化，而是**决策具有先后顺序**。

学习动态规划时，可以先问自己三个问题：

1. **问题能否拆分成多个阶段？**
    
2. **当前决策会不会影响下一阶段？**
    
3. **前面的最优结果，能否帮助后面的决策？**
    

如果这三个问题的答案都是肯定的，那么动态规划往往就是一种值得尝试的建模方法。

从旅行规划、库存管理，到能源调度、生产安排，许多现实问题都需要连续做出决策。动态规划正是通过"记录过去、优化未来"的思想，让复杂问题变得有章可循。

> **动态规划优化的不是某一步，而是整个决策过程。**
