# 📖 故事引入

很多建模竞赛选手都会遇到一个问题：

- 题目读完了
    
- 数据也看了
    
- 但不知道用什么模型
    

于是开始：

- 随便试ARIMA
    
- 或者直接上XGBoost
    
- 或者AHP + TOPSIS乱组合
    

结果是：

> ❌ 模型很高级，但方向是错的

于是一个更关键的问题出现了：

> **有没有一种方法，可以“看到题目就知道用什么模型”？**

---

# 🧠 核心思想

一句话：

> **选模型 = 识别问题类型 + 匹配方法结构**

---

# 🧭 一、第一步：判断问题类型

---

## 📌 ① 是否在问“最优解”？

关键词：

- 最大 / 最小
    
- 最优方案
    
- 如何分配
    
- 如何安排
    

👉 结论：

> ✔ 优化模型

---

## 📌 ② 是否在做“排序/评分”？

关键词：

- 综合评价
    
- 哪个更好
    
- 排名
    
- 指标体系
    

👉 结论：

> ✔ 评价模型

---

## 📌 ③ 是否在预测未来？

关键词：

- 未来
    
- 趋势
    
- 变化
    
- 预测
    

👉 结论：

> ✔ 预测模型

---

## 📌 ④ 是否在“分类/分组”？

关键词：

- 是否属于
    
- 是否风险
    
- 分群
    
- 类型划分
    

👉 结论：

> ✔ 机器学习模型

---

# 🧠 二、第二步：细分模型选择

---

# 📌 ① 优化问题怎么选？

```text
结构复杂 → 动态规划
路径问题 → 图论模型
资源分配 → 线性规划
组合爆炸 → 遗传算法 / 模拟退火
多目标 → 多目标优化
```

---

# 📌 ② 评价问题怎么选？

```text
主观权重 → AHP
客观权重 → 熵权 / CRITIC
排序 → TOPSIS
效率 → DEA
```

---

# 📌 ③ 预测问题怎么选？

```text
时间序列 → ARIMA / SARIMA
小样本 → 灰色预测
趋势+季节 → Holt-Winters
变量关系 → 回归（OLS / Logistic）
```

---

# 📌 ④ 分类问题怎么选？

```text
简单分类 → Logistic
非线性 → SVM / RF
高维数据 → XGBoost / LGBM
文本/概率 → 朴素贝叶斯
```

---

# 📌 ⑤ 聚类问题怎么选？

```text
已知K → KMeans
任意形状 → DBSCAN
结构分析 → 层次聚类
降维辅助 → PCA
```

---

# 🧠 三、第三步：多模型组合策略（竞赛核心🔥）

---

## 📌 标准组合方式

### ✔ 1. 评价 + 优化

- AHP + 线性规划
    
- TOPSIS + 规划模型
    

---

### ✔ 2. 预测 + 决策

- ARIMA + 优化模型
    

---

### ✔ 3. 多模型对比

- Logistic vs RF vs XGBoost
    

---

### ✔ 4. 集成学习

- Bagging / Boosting / Stacking
    

---

# 🧠 四、一句话决策口诀（核心🔥）

---

> ✔ 有“最优” → 优化  
> ✔ 有“排名” → 评价  
> ✔ 有“未来” → 预测  
> ✔ 有“类别” → 分类/聚类

---

# 🏆 五、完整决策流程图（核心资产）

```text
题目
 ↓
问题类型识别
 ↓
选择大类模型
 ↓
细分模型
 ↓
多模型验证
 ↓
评价体系（ROC / AUC / F1）
```

---

# 📌 本章小结

选模型不是猜，而是：

> 🧠 “问题结构 → 模型映射”

---

## 🔥 三个核心能力：

1. 判断问题类型
    
2. 匹配模型类别
    
3. 组合验证结果
    

---

# 🚀 最终一句话总结

> **建模竞赛的本质，不是选最强模型，而是选对模型。**
