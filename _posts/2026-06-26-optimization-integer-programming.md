# 📖故事引入

社区快递站最近业务增长很快，老板准备采购新的配送车辆。

市场上有两种车型可供选择：小型三轮车价格便宜、适合短距离配送；新能源面包车运力更大，但采购成本也更高。

老板已经计算好了预算，也知道每天需要满足的配送能力，但真正让他犯难的是：到底应该买多少辆？

如果按照线性规划计算，答案可能是购买 **3.7 辆面包车**。数学上这是最优解，可现实中显然没有人能买半辆车。

当决策对象必须是完整的个体时，普通线性规划就不再适用了。

今天要学习的，就是在线性规划基础上进一步发展的模型——**整数规划**。

# 🧠 什么是整数规划？

一句话来说：

> **整数规划，就是要求决策变量必须取整数的优化模型。**

在线性规划中，我们通常只关心目标函数最大或最小，而默认变量可以取任意实数。

但现实世界并不是这样。

例如：

- 采购车辆不能买 2.5 辆；
    
- 招聘员工不能招 8.3 人；
    
- 建设仓库不能建 1.6 座；
    
- 安排配送路线不能派出半个配送员。
    

这些问题都有一个共同特点：**决策变量代表的是一个个完整的对象。**

因此，在建立数学模型时，我们除了保留目标函数和各种资源约束外，还需要增加一个新的限制：

$$  
x_i\in\mathbb{Z}  
$$

也就是说，每个决策变量只能取整数。

整数规划并没有改变建模思路，它只是让数学模型更加贴近现实。

# 🚩 什么时候想到整数规划？

很多同学第一次接触整数规划时，会误以为它和线性规划是两种完全不同的模型。

其实，它们最大的区别只有一句话：

> **变量能不能出现小数。**

如果题目中出现下面这些关键词，就应该优先考虑整数规划：

- 采购多少辆车
    
- 建设多少个仓库
    
- 招聘多少名员工
    
- 安装多少台设备
    
- 配置多少台服务器
    
- 安排多少支队伍
    
- 生产多少件产品（必须为整数）
    

这类题目的决策变量，本质上都是**计数问题**。

如果求出的结果出现 4.8 辆车、6.2 名员工，即使数学上成立，现实中也无法执行。

当然，并不是所有整数问题都需要使用整数规划。

如果变量表示的是：

- 面积
    
- 长度
    
- 时间
    
- 重量
    
- 投资金额
    

这些量通常允许出现小数，此时普通线性规划往往已经足够。

因此，在建立模型之前，不妨先问自己一句：

> **题目中的决策对象，能不能被切成一半？**

如果答案是否定的，那么整数规划很可能就是正确的选择。

# 🏆 一个完整案例

某社区快递站计划采购配送车辆，提高每天的派件能力。

市场上有两种车型：

|车型|采购成本（万元/辆）|日配送能力（件/辆）|
|---|--:|--:|
|电动三轮车|4|120|
|新能源面包车|7|220|

快递站最多可投入 **35 万元**采购车辆，希望每天至少具备 **1000 件**的配送能力。

今天到底需要决定什么？

答案很简单：

- 电动三轮车购买多少辆，用变量 $x_1$ 表示；
    
- 新能源面包车购买多少辆，用变量 $x_2$ 表示。
    

接下来，还要回答另一个问题：

怎样才算最优？

老板希望**花的钱尽可能少**。

因此，目标函数可以写成：

$$  
\min Z=4x_1+7x_2  
$$

然后看看有哪些现实限制。

首先，每天至少完成 1000 件配送。

一辆三轮车每天配送 120 件，一辆面包车每天配送 220 件。

因此，总配送能力就是：

$$  
120x_1+220x_2  
$$

由于每天至少需要完成 1000 件，因此得到约束：

$$  
120x_1+220x_2\ge1000  
$$

另外，车辆数量不能是负数：

$$  
x_1\ge0,\quad x_2\ge0  
$$

最关键的一条约束来了。

因为车辆只能整辆购买，所以：

$$  
x_1,x_2\in\mathbb{Z}  
$$

把这些条件整理起来，就得到了完整模型：

$$
\begin{aligned}
\min \quad & Z = 4x_1 + 7x_2 \\
\text{s.t.} \quad & \left\{
\begin{aligned}
& 120x_1 + 220x_2 \ge 1000, \\
& x_1 \ge 0, \quad x_2 \ge 0, \\
& x_1, x_2 \in \mathbb{Z}.
\end{aligned}
\right.
\end{aligned}
$$

整个建模过程依然是在做一件事：

现实问题

↓

数学表达

例如：

- 一辆面包车每天配送 220 件 → $220x_2$
    
- 每天至少配送 1000 件 → $120x_1+220x_2\ge1000$
    
- 一辆车只能整辆购买 → $x_1,x_2\in\mathbb{Z}$
    

与线性规划相比，唯一新增的内容，就是**整数约束**。

但正是这一条约束，让模型真正能够指导现实决策。

# 💻 Python 实现

由于 `scipy.optimize.linprog()` 只能求解连续变量的线性规划，因此整数规划通常使用 **PuLP** 等优化建模库。

目标函数：

$$  
4x_1+7x_2  
$$

在 Python 中对应：

```python
# 成本系数
4 * x1 + 7 * x2
```

完整代码如下：

```python
from pulp import LpProblem, LpVariable, LpMinimize, LpInteger, lpSum, PULP_CBC_CMD

# 创建最小化问题
model = LpProblem("Vehicle_Purchase", LpMinimize)

# 定义整数变量
x1 = LpVariable("tricycle", lowBound=0, cat=LpInteger)
x2 = LpVariable("van", lowBound=0, cat=LpInteger)

# 目标函数：采购成本最小
model += 4 * x1 + 7 * x2

# 配送能力约束
model += 120 * x1 + 220 * x2 >= 1000

# 求解模型
model.solve(PULP_CBC_CMD(msg=False))

print("求解状态：", model.status)
print("三轮车数量：", x1.value())
print("面包车数量：", x2.value())
print("最低采购成本：", model.objective.value(), "万元")
```

运行后可以得到满足配送需求且采购成本最低的车辆配置方案。

与上一章相比，这段代码最大的变化就是变量的定义：

```python
cat=LpInteger
```

这一行告诉求解器：**变量只能取整数**。

数学模型中的整数约束，也就在程序中得到了体现。

# 📌 本章小结

整数规划并没有改变优化问题的基本思路，它只是进一步回答了一个现实世界必须面对的问题：

**决策变量能不能出现小数？**

学习整数规划时，可以先问自己三个问题：

1. **什么需要我决定？**
    
2. **目标是成本最低还是收益最高？**
    
3. **这些决策对象能不能被分成一半？**
    

如果第三个问题的答案是否定的，就应该考虑在线性规划的基础上加入整数约束。

从车辆采购、员工招聘，到仓库建设、设备配置，整数规划解决的都是**"数量必须是完整个体"**这一类优化问题。

> **优秀的建模，不只是找到最优解，更要找到现实中能够执行的最优解。**
