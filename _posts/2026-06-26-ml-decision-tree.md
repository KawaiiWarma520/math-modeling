# 📖 故事引入

一家电商平台想预测用户是否会购买某款商品。

他们整理了用户数据：

- 年龄
    
- 是否会员
    
- 浏览次数
    
- 是否加购
    

但问题是：

> 不同用户的行为组合非常复杂。

例如：

- 年轻 + 浏览多 → 可能会买
    
- 年长 + 会员 → 也可能会买
    
- 浏览少 + 非会员 → 大概率不会买
    

如果用一个公式（比如 Logistic 回归），虽然可以算概率，但业务人员会觉得：

> **太“黑箱”，不容易理解。**

于是他们提出一个更直观的问题：

> **能不能像人做判断一样，一步一步分情况判断？**

今天要学习的，就是解决这类问题的经典模型——**决策树**。

---

# 🧠 什么是决策树？

一句话来说：

> **决策树，就是通过一系列“如果……那么……”规则，把数据一步步分裂成不同类别的方法。**

它的本质不是公式，而是：

> **规则系统**

例如：

- 如果是会员 → 继续判断
    
- 如果浏览次数 > 10 → 可能购买
    
- 否则 → 不购买
    

---

# 🚩 什么时候想到决策树？

如果题目中出现下面这些情况，就可以考虑决策树：

- 分类问题
    
- 多条件判断
    
- 规则解释性强
    
- 希望模型可解释
    
- 数据特征较多
    
- 需要逐步决策
    

例如：

- 是否贷款通过
    
- 是否购买产品
    
- 是否违约
    
- 是否流失用户
    

一个简单判断方法是：

> **如果问题可以拆成一系列“条件判断”，就可以考虑决策树。**

---

# 🧠 决策树的核心思想

决策树做的事情非常像人类思考：

> 先问一个问题，再根据答案继续问下一个问题。

例如：

```
用户是否会员？
    ↓
是 → 浏览次数是否大于10？
           ↓
        是 → 可能购买
        否 → 继续观察

否 → 是否加购？
           ↓
        是 → 可能购买
        否 → 不购买
```

---

# 🧠 关键问题：如何决定“先问哪个问题”？

这是决策树最核心的地方：

> **每一步都要选择“最有区分能力”的特征**

也就是：

- 哪个特征能最快把数据分开？
    

---

# 📌 信息熵（核心思想）

决策树用“信息熵”衡量混乱程度：

$$  
H(S) = -\sum p_i \log p_i  
$$

如果数据越混乱：

- 熵越大
    

如果数据越纯：

- 熵越小
    

---

# 📌 信息增益（关键）

我们希望找到：

> 分裂后“最干净”的方式

信息增益：

$$  
Gain = H(原始) - H(分裂后)  
$$

👉 选择信息增益最大的特征

---

# 🏆 一个完整案例

某电商平台统计用户行为：

|用户|是否会员|浏览次数|是否购买|
|---|---|--:|---|
|A|是|12|是|
|B|否|3|否|
|C|是|8|是|
|D|否|15|是|
|E|否|2|否|

---

## 今天要解决的问题：

> 用户是否会购买？

---

## 建模过程（直觉）

### 第一步：看“会员”

会员用户：

- 大部分购买
    

👉 信息增益高

---

### 第二步：再分“浏览次数”

进一步细分

---

## 最终结构：

```
是否会员？
    ↓
是 → 购买
否 → 浏览次数 > 10？
              ↓
            是 → 购买
            否 → 不购买
```

---

# 💻 Python 实现

```python
import numpy as np
from sklearn.tree import DecisionTreeClassifier

# 特征：会员(1/0), 浏览次数
X = np.array([
    [1, 12],
    [0, 3],
    [1, 8],
    [0, 15],
    [0, 2]
])

# 标签：是否购买
y = np.array([1, 0, 1, 1, 0])

# 建立决策树
model = DecisionTreeClassifier(criterion='entropy', max_depth=3)
model.fit(X, y)

# 预测
print(model.predict([[0, 12]]))
```

# 📌 本章小结

决策树解决的是：

> **如何用“规则结构”进行分类或预测**

学习决策树时，可以记住三个关键词：

1. **分裂（一步步拆问题）**
    
2. **信息增益（选择最优拆分）**
    
3. **树结构（可解释规则）**
    

---

## 🔥 和 Logistic 的区别

|模型|特点|
|---|---|
|Logistic|数学公式，概率输出|
|决策树|规则结构，逻辑判断|

---

> **Logistic是“算出来的答案”，决策树是“一步步问出来的答案”。**
