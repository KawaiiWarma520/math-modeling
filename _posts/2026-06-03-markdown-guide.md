## 第一章 Markdown 扫盲🧹

<a id="q1什么是-markdown"></a>
### Q1：什么是 Markdown？

Markdown 是一门**轻量级标记语言**。在编辑文字时，你只需要通过按下键盘上的符号，就可以对你的文档进行排版并使其更加美观！

例如，你只需要在文本前后加两个星号，就可以实现文本加粗：

```
**加粗文本**
```

**加粗文本**

或者在文本前加一个短横和空格，就可以列出无序列表：

```
- 第一
- 第二
```

- 第一
- 第二

就是这样，是不是很简单！

所以，Markdown 语言产生的目的就是——让人们不需要手动排版和学习复杂的编程语言，就可以沉浸式地写出美观且易读的文档。

<a id="q2为什么要学习-markdown"></a>
### Q2：为什么要学习 Markdown？

相信在了解 Markdown 之前，你一定用过 Word / WPS 这样的文本编辑软件。当你在这些软件中编辑文字时，常常需要利用鼠标来回点击菜单栏才可以对其进行排版。这样做不仅麻烦，还很影响写作时的"沉浸感"，常常会因为反复调整格式而中断写作思路……

而且，如果你想在 Word / WPS 里插入数学公式，往往需要下载其他软件（如 MathType、AxMath）来协助。如果需要保存代码，直接复制粘贴又很容易导致代码格式错乱，再次粘贴到代码编辑器里运行时难免会因为缩进被删掉而报错。

更重要的是，当你和 AI 进行对话时，AI 输出的内容排版往往十分美观。而当你把 AI 生成的内容复制粘贴到 Word / WPS 里时，却发现原本的美观格式多了各种奇怪的符号，使得你不得不删掉它们或自己手打内容，非常不方便。

**而以上这些痛点，在 Markdown 里，是完全不存在的！**

- 鼠标来回调整格式很麻烦？➡️ 在 Markdown 里用键盘就可以做到！
- 插入公式和代码会乱码？➡️ 在 Markdown 里用美元符号 `$$` 和反引号 `` ``` `` 就可以做到！

比如你在建模中用到的公式和代码：

$$
\hat{\beta} = (X^T X)^{-1} X^T Y
$$

```Python
import pandas as pd
import matplotlib.pyplot as plt

df = pd.read_csv("data.csv")
plt.scatter(df["x"], df["y"])
plt.title("数据散点图")
plt.savefig("output.png")
```

并且，如果想要直接粘贴 AI 的内容，在 Markdown 里可以完美复现——因为 AI 使用的就是 Markdown 语法来对内容进行输出，而你看到的就是 Markdown 语法渲染后的内容！这也是为什么你在粘贴 AI 输出的内容时，总会看到各种符号的原因 ✅

除此之外，Markdown 的优点还有很多：学习成本低、文件极小、可读性强、生态强大（GitHub、Obsidian、Notion，写Agent的Skills都在用）等等，在后面的学习中你也会慢慢感受到 Markdown 的强大之处！

> 💡 **数模视角**：在数学建模比赛中，AI 是你最得力的助手——帮你分析数据、生成代码、推导公式、梳理建模思路。而 AI 输出的语法正是 **Markdown**！学会 Markdown，意味着你可以把 AI 生成的公式、代码、思路分析**直接复用**到自己的文档中，无需二次排版！

### 3. 链接与图片

**链接格式：**

```
[输入你想在链接上显示的文字](输入网址)
```

例如：

```
[哔哩哔哩 (゜-゜)つロ 干杯~-bilibili](https://www.bilibili.com/)
```

效果：

[哔哩哔哩 (゜-゜)つロ 干杯~-bilibili](https://www.bilibili.com/)

**图片格式：**

```
![图片名称](图片链接)
```

例如：

```
![Warma](https://tse1-mm.cn.bing.net/th/id/OIP-C.pAmmYaSyH8xeuRxJ8-p69gAAAA)
```

效果：

![Warma](https://tse1-mm.cn.bing.net/th/id/OIP-C.pAmmYaSyH8xeuRxJ8-p69gAAAA?w=157&h=180&c=7&r=0&o=7&dpr=1.3&pid=1.7&rm=3)

**链接与图片的区别：**
- 链接用 `[文字](地址)`，图片用 `![文字](地址)`
- 图片比链接多一个 `!`——可以理解为"哇，这是个图片！"（感叹号）

> ⚠️ **平台兼容提醒**：在 Obsidian 编辑器中，可以用 `![图片名|宽度x高度]` 的语法调整图片尺寸。这是 Obsidian 特有的扩展语法，在其他平台（如 CSDN、知乎）的标准 Markdown 中不支持此功能，请以标准的 `![](链接)` 格式为准。

### 4. 代码块

**行内代码：** 用单个反引号包裹 `` `代码` ``，适合在段落中提及代码变量或命令。

**代码块：**

````

```
````

例如，在数模中用模型做完数据分析后，直接把核心代码复制到 Markdown 中保存：

````
```Python
import pandas as pd

import numpy as np

# 读取数据
df = pd.read_csv("competition_data.csv")

# 计算相关系数矩阵

plt.colorbar()

plt.yticks(range(len(corr.columns)), corr.columns)
plt.title("变量相关系数热力图")
````

<a id="5-表格"></a>
### 5. 表格

```
| 模型名称 | 优点 | 缺点 |
|:---|:---|:---|
| 线性回归 | 简单直观、可解释性强 | 只能拟合线性关系 |
| 决策树 | 不需要特征缩放、可处理非线性 | 容易过拟合 |
| 随机森林 | 泛化能力强、抗过拟合 | 模型较大、训练慢 |
| SVM | 小样本效果好、泛化能力强 | 参数调优困难 |
```

效果：

| 模型名称 | 优点 | 缺点 |
|:---|:---|:---|
| 线性回归 | 简单直观、可解释性强 | 只能拟合线性关系 |
| 决策树 | 不需要特征缩放、可处理非线性 | 容易过拟合 |
| 随机森林 | 泛化能力强、抗过拟合 | 模型较大、训练慢 |
| SVM | 小样本效果好、泛化能力强 | 参数调优困难 |

**格式说明：**

表格的第二行（`|:---|---:|` 那一行）用来声明对齐方式。短横线的数量不限，但最少需要三个。

短横线前后的冒号 `:` 代表对齐方向：

| 写法 | 含义 |
|:---|:---|
| `:---` | 左对齐 |
| `:---:` | 居中对齐 |
| `---:` | 右对齐 |

> 💡 可以这样记：就像天秤上的砝码，冒号在哪边，文字往哪偏；两边一样重，文字则居中。

> 💡 **小技巧**：表格的 Markdown 语法比较复杂，很多编辑器（如 Obsidian、Typora）都提供了"鼠标右键创建表格"的功能，不妨试试看！

<a id="6-公式latex"></a>
### 6. 公式（LaTeX）

Markdown 语法中所使用的公式语言和 LaTeX 语法高度一致。原因在于 LaTeX 的公式体系非常完善，且可以扩展到物理、化学、生物，音乐等多个领域，是通用的"学术语言"之一。Markdown 选择与 LaTeX 共用公式语法，而不是"从头造轮子"。

不过 LaTeX 的公式语法相对复杂，学习成本较高。这里我推荐一个**偷懒技巧**：

> 💡 将你需要插入的数学公式**拍照**，然后把图片交给 AI 工具，说"请给我该公式的 LaTeX 语法"，再把 AI 给出的代码粘贴到 Markdown 中用美元符号包裹起来即可。

**行内公式：** 用单个 `$` 包裹

```
这是行内公式：$y = \beta_0 + \beta_1 x + \varepsilon$
```

效果：这是行内公式 $y = \beta_0 + \beta_1 x + \varepsilon$

**块级公式：** 用两个 `$$` 包裹

```
$$
\hat{\beta} = \arg\min_{\beta} \sum_{i=1}^{n} (y_i - \beta_0 - \beta_1 x_i)^2
$$
```

效果：

$$
\hat{\beta} = \arg\min_{\beta} \sum_{i=1}^{n} (y_i - \beta_0 - \beta_1 x_i)^2
$$

更多数模常用公式：

```
$$
R^2 = 1 - \frac{SS_{res}}{SS_{tot}} = 1 - \frac{\sum (y_i - \hat{y}_i)^2}{\sum (y_i - \bar{y})^2}
$$
```

效果：

$$
R^2 = 1 - \frac{SS_{res}}{SS_{tot}} = 1 - \frac{\sum (y_i - \hat{y}_i)^2}{\sum (y_i - \bar{y})^2}
$$

**注意事项：**

1. **行内公式**用前后一个美元符号 `$...$` 包裹，**块级公式**用两个 `$$...$$`
2. 如果需要插入多行公式，需要用 LaTeX 的 `begin` / `end` 环境（如 `align`、`gather`）
3. Markdown 中插入公式时不会自动编号，如需编号，可在公式结尾、`$$` 前添加 `\tag{1}` 来实现

> 💡 **数模小贴士** 🏆：在数学建模比赛中，你可以先用 Markdown 写建模思路文档，公式用 `$$` 嵌入 LaTeX。到了需要提交正式论文时，这些公式可以直接复制到 Overleaf（LaTeX 在线编辑器）中使用——语法完全一致，无需二次排版！这能让你在比赛的前期思路整理阶段省下大量时间 🕐

<a id="8-转义字符"></a>
### 8. 转义字符

如果想让 Markdown 语法符号本身显示出来（而不是被渲染），在前面加反斜杠 `\`：

```
\# 这不是标题
\*\*这不是加粗\*\*
```

效果：

\# 这不是标题
\*\*这不是加粗\*\*

> 💡 **小知识**：不只 Markdown，很多编程相关语言（如 LaTeX、Python）的转义字符都是在符号前加反斜杠哦！

### 9. 脚注

**写法：**

```
在数学建模中，最小二乘法[^1]是最基础的参数估计方法之一。
而 Breiman 在 2001 年提出的随机森林算法[^2]，至今仍是数据挖掘竞赛中的热门模型。

[^1]: 最小二乘法（Ordinary Least Squares, OLS）通过最小化残差平方和来估计回归参数，由高斯于 1809 年正式提出。
[^2]: Random Forest 由 Leo Breiman 在 2001 年发表，是集成学习 Bagging 方法的代表算法之一。
```

**效果：**

在数学建模中，最小二乘法[^1]是最基础的参数估计方法之一。而 Breiman 在 2001 年提出的随机森林算法[^2]，至今仍是数据挖掘竞赛中的热门模型。

[^1]: 最小二乘法（Ordinary Least Squares, OLS）通过最小化残差平方和来估计回归参数，由高斯于 1809 年正式提出。
[^2]: Random Forest 由 Leo Breiman 在 2001 年发表，是集成学习 Bagging 方法的代表算法之一。

> 💡 脚注默认会放到文章末尾，这点和论文中的参考文献引用很像呢！阅读时点击带数字的脚注标记即可跳转到对应说明。

<a id="10-其他"></a>
### 10. 其他

上面介绍的是 Markdown 的通用语法，**在任何编辑环境里都适用**。而有些编辑器可能会在原基础上增加一些"语法扩展"，以便让 Markdown 更好地适配编辑环境：

| 编辑器 | 扩展语法示例 |
|--------|------------|
| **Obsidian** | `[[笔记名]]` 创建笔记间链接 |
| **Typora** | `[TOC]` 自动生成目录 |
| **GitHub** | `@用户` 提及、issue 自动引用 |

另外，Markdown 还支持 HTML 语法：

```html
<center>文字</center>            <!-- 文字居中 -->
<font color="red">文字</font>   <!-- 红色文字 -->
```

但要记住一个**小原则**：HTML 相对复杂，而 Markdown 支持 HTML 是为了"填补 Markdown 覆盖不到的地方"，起到的是"查漏补缺"的作用。如果全程用 HTML 撰写，时间成本和学习成本会很高，这与 Markdown 设计的初衷是相悖的。

日常编辑时，Markdown 基础语法足以满足大部分需求；当你需要对内容做精细化调整时，再请 HTML"帮忙"会更好哦！

<a id="第三章-写在后面的话"></a>
## 第三章 写在后面的话💬

看到这里的你可能已经被 Markdown 的各种语法绕得团团转了……这很正常！这么多内容刚入手，无论是谁都很难记全。那么，我们该怎样学习并记住这些语法呢？

我推荐的学习 Markdown 的方式，不是"死记硬背法"，而是**从自己动手记笔记开始**——在使用过程中逐渐掌握这些语法，养成知识积累的好习惯。写的次数多了，自然就记住了 ✅

如果你是为数学建模比赛来学习 Markdown 的，那正好可以从**建模思路整理**开始练手——打开一个 Markdown 编辑器，写写你的模型想法、公式推导、代码片段。等比赛提交论文时，你会发现前期的 Markdown 笔记已经帮你把大部分内容准备好了，直接迁移到 LaTeX 就能用。

最后，Markdown 语言的魅力不在于"多么强大"，而在于——它通过一种简单的方式，把这些曾用于花在排版上的时间，**留给了正在思考和创作的你** ❤️

就是这样！
