## 🧠 基本概念

**一句话定义**：整数规划 = 线性规划 + 决策变量必须取整数的约束。

**三种常见类型**：

| 类型 | 变量取值 | 适用场景 |
| :--- | :--- | :--- |
| **纯整数规划** | 所有变量都是整数 | 人数、机器台数、车辆数 |
| **0-1规划** | 变量只能取 0 或 1 | 是否建厂、是否选择某方案 |
| **混合整数规划** | 部分整数 + 部分连续 | 生产数量（整数）+ 原材料用量（连续） |

**核心思想**：
- 目标函数和约束条件仍然是线性的
- 区别在于：解必须从"整数点"中找，不能是小数
- 0-1 规划是整数规划的特殊形式，用于处理"是/否"的二选一决策

## ⚠️ 核心难点

整数规划比线性规划**难求解得多**——不是因为有整数就要多算几步，而是问题的复杂度会指数级爆炸。

> 线性规划可以在多项式时间内求解，但整数规划是 **NP-hard** 问题。简单说：变量一多，计算机也可能算到天荒地老。

## 🏆 经典例子：背包问题

> 背包最多能装 10 公斤。有 4 件物品，每件只能选或不选（不能装半个）。怎样装价值最大？

| 物品 | 重量 | 价值 |
|:---|:---|:---|
| A | 3kg | 5元 |
| B | 4kg | 6元 |
| C | 5kg | 7元 |
| D | 2kg | 3元 |

**模型表达**：
- 决策变量：$x_1, x_2, x_3, x_4 \in \{0, 1\}$（1=选，0=不选）
- 目标函数：$\max Z = 5x_1 + 6x_2 + 7x_3 + 3x_4$
- 约束条件：$3x_1 + 4x_2 + 5x_3 + 2x_4 \leq 10$

**为什么不能用线性规划**：
- 线性规划可能给出"选 0.5 个 A + 0.5 个 B"的无意义解
- 整数规划保证每个物品要么全选（1），要么不选（0）

## 📊 适用场景

| 场景 | 为什么必须用整数规划 | 例子 |
| :--- | :--- | :--- |
| **人员排班** | 不能安排 3.7 个人上班 | 机场安检、医院护士排班 |
| **选址问题** | 要么建（1），要么不建（0） | 从 N 个候选位置中选 K 个 |
| **生产计划** | 不能生产 2.5 台汽车 | 工厂每日生产计划 |
| **排课/排考** | 一门课只能在一个教室 | 大学期末考试安排 |
| **切割问题** | 不能切割 0.7 根钢材 | 原材料切割优化 |
| **投资组合** | 要么投（1），要么不投（0） | 项目选择、资本预算 |

## 🛠️ 解题步骤

与线性规划的五步法相同，唯一区别在 **Step 1** 定义变量时明确整数类型：

**Step 1：定义决策变量**
- 纯整数：$x_i \geq 0$ 且为整数
- 0-1 变量：$x_i \in \{0, 1\}$
- 混合整数：部分整数，部分连续

**Step 2：确定目标函数**
- $\max (\text{或} \min) Z = c_1x_1 + c_2x_2 + ... + c_nx_n$

**Step 3：确定约束条件**
- $a_{11}x_1 + a_{12}x_2 + ... \leq (\text{或} \geq, =) b_1$

**Step 4：求解**
- 使用整数规划求解器（如 pulp 的 `cat='Integer'` 或 `cat='Binary'`）

**Step 5：解释结果**
- 输出整数解，并分析含义

## ⚠️ 注意事项

**1. 整数规划更难求解**
- 线性规划的解在可行域顶点，有高效算法
- 整数规划需要从"整数点"中搜索，计算量随变量数指数增长
- 比赛中变量过多（如 >50 个 0-1 变量）时可能求解很慢

**2. 先试线性规划，再上整数规划**
- 如果线性规划的解恰好是整数，直接使用
- 如果不是整数，再切换到整数规划

**3. 0-1 变量可以模拟"或"条件**

| 逻辑条件 | 数学表达 |
|:---|:---|
| 要么选 A，要么选 B，不能都选 | $x_A + x_B \leq 1$ |
| 选 A 的前提是必须选 B | $x_A \leq x_B$ |
| 至少选一个 | $x_A + x_B \geq 1$ |
| 恰好选一个 | $x_A + x_B = 1$ |

**4. 计算超时怎么办？**
- 减少变量数量（合并同类项、去掉冗余选项）
- 设置时间限制，接受次优解
- 改用启发式算法（遗传算法、模拟退火）求近似解

## ✏️ 论文写法

**摘要中的表述**：
> 本文建立了 0-1 整数规划模型。以总收益最大化为目标，以资源容量为约束，引入 0-1 决策变量表示是否选择各候选方案，求解得到最优选择方案为...

**模型建立部分的模板**：

> **决策变量**：设 $x_i \in \{0, 1\}$ 表示是否选择第 $i$ 个方案，其中 $x_i = 1$ 表示选择，$x_i = 0$ 表示不选择。
>
> **目标函数**：以总价值最大化为目标：
> $$\max Z = \sum_{i=1}^{n} v_i x_i$$
> 其中 $v_i$ 为第 $i$ 个方案的价值。
>
> **约束条件**：
> 1. 数量约束：$\sum_{i=1}^{n} x_i = K$（恰好选择 K 个）
> 2. 资源约束：$\sum_{i=1}^{n} w_i x_i \leq W$（总重量不超过容量）
> 3. 互斥约束：$x_A + x_B \leq 1$（A 和 B 不能同时选）

**结果分析中的亮点**：

> 由于本问题包含 20 个 0-1 决策变量，理论上存在 $2^{20} \approx 104$ 万种组合。采用分支定界算法，在 2.3 秒内求得全局最优解，验证了模型的高效性。

> 与线性规划松弛解（$Z=98.5$）相比，整数规划的最优解为 $Z=96$，两者差距仅为 2.5%，说明整数约束对目标值的影响较小，本方案具有实际可行性。

## 💻 代码实现

```python
import pulp

# 1. 创建问题实例
model = pulp.LpProblem("Knapsack", pulp.LpMaximize)

# 2. 定义 0-1 决策变量
items = ['A', 'B', 'C', 'D']
x = pulp.LpVariable.dicts('x', items, cat='Binary')  # Binary = 0-1变量

# 3. 定义目标函数
value = {'A': 5, 'B': 6, 'C': 7, 'D': 3}
model += pulp.lpSum(value[i] * x[i] for i in items), "Total_Value"

# 4. 添加约束条件（重量限制）
weight = {'A': 3, 'B': 4, 'C': 5, 'D': 2}
model += pulp.lpSum(weight[i] * x[i] for i in items) <= 10, "Weight_Limit"

# 5. 求解
model.solve()

# 6. 输出结果
print("状态:", pulp.LpStatus[model.status])
print("选择方案:")
for i in items:
    if x[i].varValue == 1:
        print(f"  选物品 {i}: 价值{value[i]}, 重量{weight[i]}")
print(f"总价值 = {pulp.value(model.objective)}")
```

## 🔗 关联模型

| 模型 | 关系 | 何时使用 |
| :--- | :--- | :--- |
| **线性规划** | 整数规划的"连续版" | 变量可以是小数时 |
| **0-1规划** | 整数规划的特例 | 做"是/否"决策时 |
| **混合整数规划** | 整数规划的扩展 | 部分整数、部分连续时 |
| **多目标规划** | 目标函数的扩展 | 需要同时优化多个冲突目标时 |
| **背包问题** | 整数规划的标准形式 | 资源有限下的选择问题 |
| **指派问题** | 0-1规划的特殊形式 | 任务分配、人岗匹配 |
